Understanding the Format

The data format is simple: WIDTHxHEIGHT:RLE:DATA

  • 8x8 - Canvas size (8 pixels wide √ó 8 pixels tall)
  • RLE - Run-Length Encoding compression (optional)
  • Data - Base64 characters (0-9, A-Z, a-z, +, /) representing 64 colors

Without compression, the format is: WIDTHxHEIGHT:DATA

‚ö†Ô∏è Important: Always use the export function in the editor to generate compressed strings. The RLE format uses COUNT+CHAR encoding (e.g., "30" = three zeros) which must be consistently formatted.

JavaScript API

// Create element dynamically
const sprite = document.createElement('inline-px');
sprite.setAttribute('data', '8x8:RLE:80102120211021102021102113211011151310111513111021132114103116104116104110211420211410211410');
sprite.setAttribute('scale', '4');
sprite.setAttribute('alt', 'Heart icon');
document.body.appendChild(sprite);

// Update dynamically (e.g., animation frames)
let frame = 0;
const frames = [
    '8x8:RLE:80102120211021102021102113211011151310111513111021132114103116104116104110211420211410211410',
    '8x8:RLE:...' // Different frame
];

setInterval(() => {
    sprite.setAttribute('data', frames[frame % frames.length]);
    frame++;
}, 200);

// Change scale on click
sprite.addEventListener('click', () => {
    const currentScale = parseInt(sprite.getAttribute('scale'));
    sprite.setAttribute('scale', currentScale === 4 ? 8 : 4);
});

Batch Loading with Fetch

// Load sprite library from JSON
fetch('sprites.json')
    .then(r => r.json())
    .then(sprites => {
        Object.entries(sprites).forEach(([name, data]) => {
            const sprite = document.createElement('inline-px');
            sprite.setAttribute('data', data);
            sprite.setAttribute('scale', '4');
            sprite.setAttribute('id', name);
            document.getElementById('sprite-container').appendChild(sprite);
        });
    });

Tips & Best Practices

Optimization Tips

  • Use RLE compression - Can reduce size by 50-80% for sprites with solid areas
  • Reuse colors - The 64-color palette is optimized for small file size
  • Keep sprites small - 8√ó8 to 32√ó32 works best for ultra-compact storage
  • Cache in localStorage - Store sprite data client-side for instant loading
  • Use sprite sheets - Combine multiple sprites in one larger canvas

Accessibility

    scale="4"

Performance

  • Lazy load - Use Intersection Observer for off-screen sprites
  • Reuse canvases - Access shadowRoot canvas for multiple uses
  • Avoid excessive scale - Use CSS transform for temporary scaling
  • Preload critical sprites - Include them in initial HTML
